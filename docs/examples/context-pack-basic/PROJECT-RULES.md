# Project Rules

- Do not commit secrets, local paths, private examples, generated caches, or screenshots with private data.
- Do not bypass authentication, authorization, workspace allowlisting, or audit boundaries.
- Do not store private/runtime prompts, model outputs, auth headers, handoff tokens, passwords, API tokens, or repository credentials in receipts or logs.
- Keep Context Packs as Markdown operating packages only.
- Context Packs are not Codex skills, executable workflows, schedulers, or governance runtimes.
